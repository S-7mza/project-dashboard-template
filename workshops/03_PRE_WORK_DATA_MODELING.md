Data Modeling / Semantic Layer — Pre-work Checklist

Send 3–5 working days before the session.

Required inputs
- Completed outputs from Data Discovery: schemas, sample extracts, row counts, cardinalities.
- Proposed canonical definitions for primary entities (e.g., customer, account).
- Current questions/ambiguities about keys, grain, and slowly changing dimensions.

Required attendees
- BI lead, data modeler, ETL engineer, data owner, product owner (optional).

Attachments to include in invite
- `deliverables/DATA_INVENTORY.md`
- Draft `deliverables/KPI_SPEC.md` entries

Goals for the session
- Agree canonical tables, primary keys and grain.
- Define dimensions and facts, and simple transformation rules.
- Identify modeling anti-patterns to avoid for performance.

Post-meeting outputs
- Draft model diagram and agreed grain definitions.
- Owner-assigned tasks for building canonical tables and transformations.
